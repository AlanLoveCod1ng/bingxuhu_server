Description

Given an array of non-negative integers nums, you are initially positioned at the first index of the array.

Each element in the array represents your maximum jump length at that position.

Your goal is to reach the last index in the minimum number of jumps.

You can assume that you can always reach the last index.

Example

Input: nums = [2,3,1,1,4]
Output: 2
Explanation: The minimum number of jumps to reach the last index is 2. Jump 1 step from index 0 to 1, then 3 steps to the last index.

My Intuition

This problem can be thought through two ways, one is dynamic programming, another is greedy algorithm. I came up with the dynamic programming but not greedy. Firstly, we can just jump to the optimal solution, that is, the farthest position allowed by our current position.

[2,3,1,1,4] // if we are at 2, the farthest position we can jump to is the first 1,
            // if we do that, we would miss the best path 2,3,4

so we can’t jump rudely, we need to either record or take greedy stragegy.

Dynamic Programming

This my first solution which is not provided by LeetCode, it could pass the test but not performing well on time and space complexity.

Intuition

Since that we can’t rudely jump to the optimal solution, we need to access to every possible solution to ensure that we won’t miss the global optimal. We can do DFS to every possible solution to build a path tree to generate all the possible path, but that’s unnecessary and not what we want.

When we are iterating all the possible next position from our current position, we can record the minimum number of steps so far. So we need an array, which has the same length with original array, but the value is the minimum number of steps from the 0 index.

original: [2,3,1,1,4]
record:   [0,1,1,2,2]
2->2: 0
2->3: 1
2->1st 1: 1
2->3->2nd 1: 2
2->3->4: 2

every time we access the possible next position, we would compare the record[current]+1 with its original value and take the minimum one. Note: we need to consider that, if the position is never reached before, its value is 0, that’s exception.

After iterating all the position, we just return the record[record.length-1] which is the minimum number of steps in the last index.

Here is the sample code for java.

class Solution {
    public int jump(int[] nums) {
        int [] record = new int[nums.length];
        for(int i = 0; i<nums.length; i++){
						//iterate every possible position from current one
            for(int j = i+1; j-i <= nums[i]&&j<nums.length; j++)
                //0 exception
								if(record[j]==0){
                    record[j]=record[i]+1;
                }
                record[j] = Math.min(record[i]+1,record[j]);
            }
        }
        return record[record.length-1];
    }
}

Time complexity: O(n^2), which is bad

Greedy Algorithm

Greedy algorithm is much better than dynamic programming but harder to understand. The key is to understand the Greedy Strategy.

Greedy Strategy

The greedy strategy is that, when we are deciding what is our next position, we can jump to the farthest one, local optimal, which is discussed before, that ain’t work. But we can jump to the position which give us the farthest possible.

[3,5,2,1,4,1,6,2]
now we are at 3, we have three posible next position, 5, 2, 1, which one?
if we jump to the local optimal, that is 1.
Considering the greedy strategy, we jump to 5 because 5 provide us farthest possible
that is, 1+5=6, so we decide to jump to 5.

But, how to verify this greedy algorithm, why it’s right?

We can prove that via every step, for example, when we are at 3. Jumping to 5 provide us a subarray, which is next possible position

[3,5,2,1,5,1,6,2]
5 provide [2,1,5,1,6]
2 provide [1,5]
1 provide [5]

image from leetcode

so, in every iteration, position selected by greedy algorithm always provide us longest subarray, that is, most position to detect, choosing this one will provide us global optimal finally.

Here is code for java.

class Solution {
    public int jump(int[] nums) {
        int jumps = 0, currentJumpEnd = 0, farthest = 0;
        for (int i = 0; i < nums.length - 1; i++) {
            // we continuously find the how far we can reach in the current jump
            farthest = Math.max(farthest, i + nums[i]);
            // if we have come to the end of the current jump,
            // we need to make another jump
            if (i == currentJumpEnd) {
                jumps++;
                currentJumpEnd = farthest;
            }
        }
        return jumps;
    }
}
